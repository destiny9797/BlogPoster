栈和队列

栈的应用

匹配问题（括号匹配、对对碰）

队列的应用

滑动窗口，寻找滑动窗口中的最大值，用双端队列来维护一个单调队列，队头总是最大值

单调队列的思想：维护一个单调递减的数列，例如，如果要加入的数大于队尾的数，则把队尾的数不断弹出，直到队尾不大于要加入的数；如果要删除的数等于队头的数（即最大的数），则把队头弹出。

优先队列：实质是个堆，用堆排序维护，每次push或者pop都会重新排序，优先级最高的在top

定义优先队列的方法：——是和sort相反的

priority_queue<int> q;//默认是从大到小

priority_queue<int, vector<int> ,less<int> >q;//从大到小排序

priority_queue<int, vector<int>, greater<int> >q;//从小到大排序

优先队列自定义比较的几种写法：

//方式一：重载()
struct Node
{
	char data;
	int weight;
	Node(char _data, int _weight)
	{
		data = _data;
		weight = _weight;
	}
};
struct cmp1
{
	bool operator () (const Node &a, const Node &b)		  
	{
		//大的优先级高
		return a.weight < b.weight;			
	}
};

priority_queue< Node, vector<Node>, cmp1> q1;


//方式二：重载<
struct Node2 
{
	char data;
	int weight;
	Node2(char _data, int _weight)
	{
		data = _data;
		weight = _weight;
	}
	friend bool operator < (const Node2 &a,const Node2 &b)
	{
		return a.weight < b.weight; 		
	} 
};
priority_queue<Node2> q3;

题外话（栈和队列的底层实现）

「栈是以底层容器完成其所有的工作，对外提供统一的接口，底层容器是可插拔的（也就是说我们可以控制使用哪种容器来实现栈的功能）。」

所以STL中栈往往不被归类为容器，而被归类为container adapter（容器适配器）。

「我们常用的SGI STL，如果没有指定底层实现的话，默认是以deque为缺省情况下栈的低层结构。」

deque是一个双向队列，只要封住一段，只开通另一端就可以实现栈的逻辑了。

「SGI STL中 队列底层实现缺省情况下一样使用deque实现的。」
