IP

MAC是实现直连的两个设备之间通信，而IP是在没有直连的网络中实现点对点通信。在网络中数据包传输时，源IP地址和目标IP地址一直不变，但源MAC地址和目标MAC地址一直在变化。

IP地址是以网卡来配置的，像服务器、路由器等有2个以上网卡，则有2个以上的IP地址。

IP地址的基础

IP分类地址

类别号+网络号+主机号

IP地址中主机号全1表示网络下的所有主机，用于广播；全0指定这个网络。

无分类地址CIDR——没有分类

网络号+主机号

10.100.122.2/24，表示前24位是网络号，剩余8位是主机号。子网掩码是255.255.255.0，子网掩码和IP地址求与，则可以得到网络号。

子网掩码还可以有划分子网的作用。例如在C类地址中，使用子网掩码255.255.255.192对其进行子网划分，则可以从8位主机号中借用2位作为子网号。

IP分片

以太网的MTU（最大传输单元）是1500字节，当IP数据包大于MTU是，源主机会分片，目标主机再进行重组。

IPv6

128位，更多的IP地址，并且

  • 便捷：可自动配置，即使没有DHCP服务器
  • 传输性能：简化头部，减轻路由器负荷
  • 安全性

IP协议相关技术

DNS域名解析

将域名网址转换为具体的IP地址。

客户端只要找到任意一台DNS服务器，就可以通过它找到根据DNS服务器，再一路顺藤摸瓜找到位于下层的某台目标DNS服务器。

ARP

由IP地址得到MAC地址。

RARP则是由MAC地址得到IP地址，经常用于将打印机等小型嵌入式设备接入网络时。

DHCP

动态获取IP地址

DHCP客户端进程监听68端口号，DHCP服务端进程监听67端口号。

DHCP交互中，都是使用UDP广播通信。

NAT网络地址转换

同个公司、家庭、教室内的主机对外部通信时，把私有IP地址转换成公有IP地址。

如何解决IPv4地址耗尽的问题呢？——NAPT，即网络地址和端口一起转换

NAT穿透技术：客户端主动从 NAT 设备获取公有 IP 地址，然后⾃自⼰己建⽴立端⼝口映射条⽬目，然后⽤用这个条⽬目 对外通信，就不不需要 NAT 设备来进⾏行行转换了。——解决外部无法主动与NAT内部服务器建立连接、NAT路由器重启、转换开销等的问题。

ICMP互联网控制报文协议：

确认 IP 包是否成功送达目标地址、报告发送过程中 IP 包被废弃的原因和改善网络设置等。ICMP负责通知失败的原因。

几种类型

  • 0和8用于判断所发送的数据包是否成功到达对端

    ping命令中就是用的0和8。

  • 3目标不可达
    • 网络不可达0
    • 主机不可达1
    • 协议不可达2
    • 端口不可达3
    • 需要分片但设置了不分片4
  • 4原点抑制：缓和网络拥堵
  • 5重定向：如果路由器发现发送端主机使⽤了「不是最优」的路径发送数据，那么它会返回⼀一个 ICMP 重定向消息给这个主机。
  • 11超时：IP包中的TTL（生存周期），每经过一次路由器就会减1。
